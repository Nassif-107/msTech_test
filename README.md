# ZenPulse: AI Meditation App

Прототип мобильного приложения для медитаций с AI-генерацией аффирмаций.

## Стек

- **React Native + Expo SDK 54**
- **expo-router** — файловая маршрутизация
- **NativeWind v4** — стилизация через Tailwind CSS
- **TypeScript** — строгая типизация
- **AsyncStorage** — сохранение статуса подписки

## Запуск

```bash
npm install
npx expo start
```

Откройте приложение через Expo Go на телефоне или в эмуляторе.

## Структура проекта

```
src/
├── app/                    # Экраны (expo-router)
│   ├── _layout.tsx         # Корневой layout
│   ├── index.tsx           # Редирект
│   ├── paywall.tsx         # Экран подписки
│   └── (tabs)/             # Табы
│       └── meditations.tsx # Главный экран
├── components/             # Компоненты
│   ├── paywall/            # Компоненты Paywall
│   └── meditation/         # Компоненты медитаций и AI
├── context/                # React Context (подписка)
├── constants/              # Данные, тема, аффирмации
├── hooks/                  # Хуки (useSubscription, useAffirmation)
└── types/                  # TypeScript типы
```

## Функционал

1. **Paywall** — экран подписки с выбором тарифа и кнопкой «Попробовать бесплатно»
2. **Медитации** — список сессий с логикой блокировки (серые карточки → Paywall)
3. **AI Настрой дня** — выбор настроения из 3 эмодзи → генерация аффирмации

## Контрольный вопрос

**«С какими специфическими проблемами мобильной верстки ИИ справляется хуже всего и как ты контролировал его работу, чтобы приложение не сломалось на маленьких экранах?»**

Основные проблемы мобильной верстки, с которыми ИИ справляется хуже всего:

1. **SafeArea** — ИИ часто забывает о «чёлке» и home indicator. Решение: явные инструкции в каждом промпте + использование `useSafeAreaInsets()` на каждом экране с ручной проверкой отступов.

2. **Переполнение на маленьких экранах** — на iPhone SE текст и карточки могут не вмещаться. ИИ не учитывает реальные размеры экрана. Решение: использование `ScrollView` вместо жёстких высот, относительных размеров (`flex-1`, процентные отступы), и проверка в эмуляторе на разных устройствах.

3. **Touch targets** — ИИ делает кнопки слишком маленькими. Решение: минимальный размер кликабельных элементов 44pt, `Pressable` с достаточным padding.

4. **Фиксированные размеры шрифтов** — ИИ ставит пиксельные размеры, которые могут быть нечитаемыми на SE. Решение: тестирование на минимальном экране и корректировка.

Контроль работы ИИ: визуальная проверка в эмуляторе на iPhone SE и Pro Max после каждого этапа, явные требования к SafeArea и отзывчивости в промптах, использование CLAUDE.md с правилами проекта.
