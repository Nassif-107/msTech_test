# ZenPulse: AI Meditation App

## Как ИИ справился с мобильной спецификой

С навигацией ИИ справился хорошо, expo-router с файловой маршрутизацией настроился правильно, Stack для Paywall и Tabs для медитаций заработали сразу. Переход из заблокированной карточки на Paywall и обратно после «покупки» работает корректно.

SafeArea ИИ добавил сам без напоминаний,  на обоих экранах используется `useSafeAreaInsets()` с отступами через `paddingTop: insets.top` и `paddingBottom: insets.bottom`. Тут проблем не было.

Где были реальные проблемы, это совместимость пакетов. ИИ изначально поставил React 18.3.1 и React Native 0.76.9, а Expo SDK 54 ожидал React 19.1.0 и RN 0.81.5. Из-за этого приложение крашилось с ошибкой `TurboModuleRegistry.getEnforcing('PlatformConstants')`. Пришлось вручную обновлять версии. Ещё не хватало `react-native-worklets` , бандлер падал, пока не доставили пакет и не добавили `react-native-reanimated/plugin` в babel конфиг. NativeWind тоже не встал с первого раза из-за конфликта peer-зависимостей.

## Контрольный вопрос

**«С какими специфическими проблемами мобильной верстки ИИ справляется хуже всего и как ты контролировал его работу, чтобы приложение не сломалось на маленьких экранах (iPhone SE vs Pro Max)?»**

Главная проблема - ИИ плохо разбирается в совместимости версий пакетов. В нашем случае он подобрал React и React Native не тех версий для Expo SDK 54, и приложение вообще не запускалось. Пришлось вручную разбираться с ошибками и обновлять зависимости. Это не совсем верстка, но без работающего приложения до верстки даже не дойдёшь.

По самой верстке - ИИ нормально использует flex-layout и ScrollView, так что на разных экранах контент не обрезается. SafeArea он добавил сам. Но он не видит результат — работает вслепую. Поэтому если что-то визуально «поехало», узнать об этом можно только открыв приложение на устройстве.

Как я контролировал:

- Написал CLAUDE.md с правилами проекта до начала - SafeArea обязательна, адаптивная верстка, никаких фиксированных высот.
- После каждого этапа запускал приложение в Expo Go и проверял на телефоне.
- Когда были ошибки сборки (PlatformConstants, worklets) - описывал ИИ ошибку, и он находил решение.
- Использовал `ScrollView` вместо жёстких контейнеров и относительные размеры через Tailwind, чтобы контент адаптировался.
